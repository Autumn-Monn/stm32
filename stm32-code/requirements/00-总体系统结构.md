## 3️⃣ 光敏电阻（关键点）

| ------- | ----- |

| RX      | TX    |

## ⚠️关键点（不注意会烧）

### 1. 供电

### 2. 电平问题

- ESP → STM32：3.3V ✔

👉 所以不用电平转换（这点你是安全的）

## 每个元件的作用（你答辩可以这样说）

### 1️⃣ 1kΩ电阻（核心）

### 3️⃣ 二极管 1N4007（不能省）

- 防止继电器断电时产生反向高压

# 三、三种方案对比（重点）

## ✔ 方案1：单I2C总线（推荐）

```
PB6 ─ SCL ─ OLED + EEPROMPB7 ─ SDA ─ OLED + EEPROM
```

### 优点：

- 最简单
- 最稳定
- 教师最认可（标准工程做法）
- 符合I2C多设备设计理念

### 缺点：

- 需要不同地址（但你这个本来就有）

👉 实际工程里99%都是这样做

---

## ⚠️ 方案2：双硬件I2C（你想做的）

```
I2C1 → OLEDI2C2 → AT24C02
```

### 引脚：

| 功能     | 引脚 |
| -------- | ---- |
| I2C1_SCL | PB6  |
| I2C1_SDA | PB7  |
| I2C2_SCL | PB10 |
| I2C2_SDA | PB11 |

---

### 优点：

- 逻辑上“干净隔离”
- 软件上简单一点点

---

### 缺点（重点）：

- 多占2个IO
- PCB更复杂
- 没必要（这是最大问题）
- 老师可能反问：**“为什么要分两条？”**

---

## ❌ 方案3：软件I2C + 硬件I2C（不推荐）

- OLED用硬件
- EEPROM用模拟I2C

👉 工程上不规范，容易被挑刺

---
