# 实施记录目录说明

这个目录用于保存后续每一个开发阶段的过程记录。

建议命名方式：

- `阶段01-GPIO点灯.md`
- `阶段02-按键轮询.md`
- `阶段03-EXTI中断.md`
- `阶段04-蜂鸣器与继电器.md`
- `阶段05-UART调试.md`
- `阶段06-ADC土壤湿度.md`
- `阶段07-I2C外设.md`
- `阶段08-DS18B20温度采集.md`
- `阶段09-联动控制逻辑.md`
- `阶段10-系统集成联调.md`

每份记录建议固定包含以下内容：

## 本阶段目标

一句话描述本次只要完成什么。

## CubeMX 配置

记录引脚模式、外设参数、NVIC 配置等。

## AI 指令

保存本次实际使用的提问语句，方便后续复现。

## 代码放置位置

说明代码写在了哪些 `USER CODE BEGIN/END` 区域，或者新增了哪些 `app_*.c/.h` 文件。

## 编写逻辑

用自己的话记录为什么这样写，不必追求很正式，但要能让未来的自己看懂。

## 问题与修正

记录调试中踩过的坑，例如：

- 电平有效方向写反
- 中断触发沿选错
- 串口波特率不一致
- 按键抖动

## 验收结果

记录最终看到的现象，以及是否达到本阶段目标。
