# 阶段02：按键轮询

## 本阶段目标

完成 4 个按键的轮询扫描与软件消抖，建立“按下事件”与“稳定按下状态”的基础输入能力，并通过按键控制 LED 验证阶段2结果。

## CubeMX 配置

- `PB12` 配置为 `GPIO_Input`，标签 `KEY_1`，`GPIO_PULLUP`
- `PB13` 配置为 `GPIO_Input`，标签 `KEY_2`，`GPIO_PULLUP`
- `PB14` 配置为 `GPIO_Input`，标签 `KEY_3`，`GPIO_PULLUP`
- `PB15` 配置为 `GPIO_Input`，标签 `KEY_4`，`GPIO_PULLUP`
- 按键接法假设为：按下接地，松开由内部上拉保持高电平
- 阶段2未启用 EXTI，中断方案留到阶段3实现

## AI 指令

- 新增 `key.h/key.c` 模块，命名风格与 `led.h/led.c` 保持一致
- 实现 4 键轮询、10ms 扫描周期、30ms 软件消抖
- 仅支持按下事件，不实现长按、双击和连发
- `main.c` 中只保留初始化与主循环调度入口
- 通过按键控制 LED 完成阶段2验收

## 编写逻辑

- 使用 `HAL_GetTick()` 作为时间基准，每 10ms 执行一次按键扫描
- 每个按键维护原始状态、稳定状态、最近变化时间和按下事件挂起标志
- 当原始状态稳定 30ms 后，更新稳定状态；如果新状态为按下，则挂起一次按下事件
- `key_get_event()` 按固定顺序返回单个事件并清除挂起标志
- `key_stage2_demo()` 消费事件并执行 LED 控制映射：
  - `KEY_1` 翻转绿灯
  - `KEY_2` 翻转红灯
  - `KEY_3` 翻转蓝灯
  - `KEY_4` 关闭全部 LED

## 问题与修正

- 当前阶段不在 `key.c` 中使用 `HAL_Delay()`，避免阻塞主循环
- 当前阶段不使用 EXTI，避免和阶段3的中断学习目标混在一起
- 板载 LED 与外接 LED 的有效电平不同，按键模块不直接写 GPIO，而是统一调用 `led` 模块接口

## 验收结果

- 上电后按键松开状态稳定，无明显误触发
- `KEY_1`、`KEY_2`、`KEY_3` 可分别稳定翻转绿灯、红灯、蓝灯
- `KEY_4` 可关闭全部 LED
- 长按按键不会重复触发多个动作
- 工程可编译、可下载、可运行，阶段2具备进入阶段3的基础
